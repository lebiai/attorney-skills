# Attorney Skills — 律师案件分析与诉讼策略工具

办一个案子，最怕什么？

**案子的走向心里没底。**

这个工具帮您把两件事做扎实：**把案子看透，把策略想清楚。**

---

## 案件深度分析

拿一份判决书扔给 AI，自动输出 10 个维度的分析报告：

| 维度 | 看什么 |
|------|--------|
| 案件基本信息 | 法院、案由、审判级别、时长 |
| 争议焦点 | 双方到底在争什么 |
| 诉讼动机 | 为什么起诉、为什么这么辩 |
| 证据体系 | 双方的证据分布、法院采信了谁 |
| 法律路径 | 法院用了哪些法条、怎么推理的 |
| 推理链条 | 法院的逻辑是怎么走的 |
| 关键转折点 | 什么证据或事实改变了结果 |
| 法官风格 | 这个法官更看重什么 |
| 规律总结 | 同类型案子有什么共同点 |
| 实战启示 | 换了您来打，可以从中学到什么 |

**什么场景用：**
- 拿到一份判决书，想快速吃透
- 研究某个法官的裁判风格
- 对比多个同类案件找规律
- 准备上诉时分析一审的问题

## 诉讼策略分析

**这是核心功能。**

您不需要上传判决书。您只需要说：

> "我有个离婚案子，男方婚前买房，婚后一起还贷，现在离婚女方要分 60%。怎么打？"

工具会做三件事：

1. **检索知识库** — 从历史判例中找出最接近的案子（来源：中国裁判文书网 + 您自己积累的案例）
2. **输出策略** — 基于真实判决结果，告诉您：
   - 类似案子的判决结果是什么
   - 法院考量的关键因素有哪些
   - 哪种诉讼路径胜算更高
   - 对方可能的抗辩策略
3. **明确告知依据** — 知识库里有的就列依据，没有的就说明没有，不会瞎编

**什么场景用：**
- 接到一个新案子，先摸底
- 做庭前准备，预判对方的牌
- 给当事人做案情分析时提供数据支撑
- 不确定某个诉讼请求有没有胜算

## 两个工具怎么配合

```
收到判决书 → 案件深度分析（吃透材料）
           → 诉讼策略分析（看同类案子怎么判的）
           → 输出口播文案（legal-script-kit）
```

三步走完，一个案子的价值就全部榨干了。

---

## 知识库怎么来的

**公共知识库**：从中国裁判文书网精选的判决书，经过结构化整理（当前 104 条，持续更新）。下载到本机后离线使用。

**私人知识库**：您每次分析过的案子，可以手动保存到本地。越用越准。

**关键原则**：知识库里没有的案子，工具会直接告诉您没有，不编造判例。

---

## 关于数据

- 所有分析在本机完成，不上传服务器
- 知识库下载后离线使用
- 私人知识库仅保存在本机，您自己控制
- 知识库更新时，只检查版本号，不涉及使用数据

## 安装

### 方式一：Codex CLI

> "安装 lawyer-case-analysis skill"
> "安装 lawyer-strategy skill"

### 方式二：手动

```bash
git clone git@github.com:lebiai/attorney-skills.git
cp -r attorney-skills/skill-lawyer-case-analysis ~/.codex/skills/
cp -r attorney-skills/skill-lawyer-strategy ~/.codex/skills/
```

首次使用策略分析时，会自动下载知识库（约 7MB，几秒钟）。
