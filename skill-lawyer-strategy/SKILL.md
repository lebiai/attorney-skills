# lawyer-strategy — 诉讼策略分析

## 触发条件

- 律师口头描述了一个案件的案情（不是判决书原文，是人话）
- 并请求策略建议，如"帮我看看这个案子怎么打"、"有什么策略"、"以前这种案子怎么判的"

## 工作流

```
Step 1: 运行 setup_knowledge.py
        → 检测/下载/更新公共知识库 distilled.db

Step 2: 读取 user_profile.json 和 personal.db
        → 了解律师的偏好领域和已积累的案例

Step 3: 将案情描述转为精准法律关键词（不要直接用用户的原话）
        规则：
        · 提取3-5个核心法律词汇（案由+事实特征+争议焦点）
        · 去掉无意义词："我有个"、"帮我看看"、"这种案子"等
        · 拆分复合词：如"婚前买房"→"婚前购房"、"婚后还贷"→"共同还贷"
        · 保留案由关键词：如"离婚""离婚后财产纠纷""借贷"
        · 示例：
          用户说："我有个离婚案子，男方婚前买房婚后一起还贷"
          → 关键词：婚前购房 婚后共同还贷 离婚 财产分割

Step 4: 运行 search_knowledge.py [关键词1 关键词2 关键词3]
        → 通道1: 公共库搜索 → top-5
        → 通道2: 私人库搜索 → top-2
        → 合并，按相关度排序

Step 5: 解析 search_knowledge.py 输出结果

        输出格式：案号|案由|法院|判决结果|案情概要|相关度|来源

        返回码：
        · 正常输出 → 有相关匹配（相关度 >= 5）
        · NO_EXACT_MATCH → 无精确匹配，但仍返回了最接近的结果
        · NO_RESULTS → 完全无匹配

Step 6: 根据结果判断：

        有相关匹配（相关度 >= 5）：
        用检索到的案例进行分析，每个策略点标注参考案号

        NO_EXACT_MATCH：
        检索到了一些接近但不完全匹配的案例。
        分析时说明"当前知识库中未找到该事实模式的精确匹配，
        以下为最接近的案例参考："
        列出结果供参考，明确标注"以下案例不完全匹配"

        NO_RESULTS：
        输出"暂无高度相似判例，建议结合具体案情咨询专业律师"
        禁止编造
```

## 分析框架

### 裁判倾向分析
- 各案例的判决结果分布（支持原告/驳回/部分支持各多少例）
- 关键影响因素（什么事实影响了胜负）
- 裁判倾向总结

### 攻防策略
- 原告策略：哪些有效 / 哪些是失误
- 被告策略：哪些抗辩被采信 / 哪些没被采信
- 双方可改进之处

### 建议方案
- 基于判例的应诉策略建议
- 区分：如果是被告方→防守策略；如果是原告方→进攻策略
- 标注每个建议的参考案号

## 输出模板

```
═══════════════════════════════════════
  诉讼策略分析报告
═══════════════════════════════════════

📋 案情概要
  [回述用户描述的案情，确认理解无误]

🔍 相似判例检索
  共找到 [N] 个相似案例：
  1. [案号] | [案由] | [法院] | [判决结果] | [相关度]
     争议焦点：[摘要]（来源：[公共库/私人库]）
  2. ...

  如无精确匹配，在此处加上：
  ⚠️ 当前知识库中未找到该事实模式的精确匹配，
     以下为最接近的案例参考，请注意甄别适用性。

📊 裁判倾向分析
  在 [N] 个相似案例中：
  · 支持原告：[N] 例
  · 驳回原告：[N] 例
  · 部分支持：[N] 例
  关键影响因素：[分析什么因素影响了判决结果]
  裁判倾向总结：[结论]

⚔️ 攻防策略（基于真实判例）
  原告策略分析：
  · 有效动作：[带案号]
  · 失误：[带案号]
  被告策略分析：
  · 有效抗辩：[带案号]
  · 失误：[带案号]
  可借鉴之处：[综合]

💡 建议方案
  [基于判例分析的应诉建议]

📎 参考案号
  [案号列表，供律师查原文]
```

## 核心约束（4条硬规则）

1. **所有策略必须基于检索到的真实判例** — 不得编造策略建议。每个策略点必须标注参考案号。
2. **NO_RESULTS 时禁止编造** — 输出"暂无高度相似判例，建议结合具体案情咨询专业律师"。
3. **NO_EXACT_MATCH 时标注不确定性** — 明确告知用户没有精确匹配，用 ⚠️ 标注。
4. **禁止绝对性承诺** — 不得输出"一定能胜诉""法院会支持"。应使用"从历史判例看，相似事实法院倾向于……"。

## user_profile 适配

如 `data/user_profile.json` 存在，读取后注意：
- `common_case_types` — 若当前案由是律师常见领域，分析时可参考历史案例数量
- `strategy_preferences.output_style` — 按律师偏好的详细程度输出
- `personal.db` 中的案例数量 — 如果私人库足够丰富（>20条），检索时私人库权重提高

## 使用方式

```bash
# 确保知识库就绪
python3 scripts/setup_knowledge.py

# 检索相似案例
python3 scripts/search_knowledge.py 婚前购房 婚后共同还贷 财产分割
```
