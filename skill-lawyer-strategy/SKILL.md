# lawyer-strategy — 诉讼策略分析

## 触发条件

- 律师口头描述了一个案件的案情（不是判决书原文，是人话）
- 并请求策略建议，如"帮我看看这个案子怎么打"、"有什么策略"、"以前这种案子怎么判的"

## 工作流

```
Step 1: 运行 setup_knowledge.py
        → 检测/下载/更新公共知识库 distilled.db

Step 2: 读取 user_profile.json 和 personal.db
        → 了解律师的偏好领域和已积累的案例

Step 3: 将律师的案情描述转为 3-5 个法律关键词
        LLM 自己完成：将口语转成法言法语检索词
        例："男方婚前买房婚后一起还贷"
        → "婚前购房 婚后共同还贷 夫妻共同财产"

Step 4: 运行 search_knowledge.py [关键词1 关键词2 ...]
        → 先为 FTS5 做空白替换：将所有空格替换为 OR
        → 通道1: 公共库 FTS5 检索 → top-5
        → 通道2: 私人库 FTS5 检索 → top-2
        → 合并排序输出
        → 无结果时输出 "NO_RESULTS"

Step 5: 根据检索结果判断：
        有结果 → 按分析框架输出策略报告
        无结果 → 明确告知"暂无高度相似判例"，结束
```

## 分析框架

### 裁判倾向分析
基于检索到的相似判例，分析：
- 各案例的判决结果分布（支持原告/驳回/部分支持各多少例）
- 什么因素影响了胜负（如：有没有书面证据、有没有婚前公证、对方是否有明显过失）
- 裁判倾向总结

### 攻防策略
逐条从判例中提取：
- 原告做了什么策略 / 哪些有效 / 哪些是失误
- 被告做了什么抗辩 / 哪些被采信 / 哪些没被采信
- 双方错过了什么机会

### 建议方案
综合以上分析，给出应诉策略建议。不是通用建议，而是基于判例中真实案情的建议。

  2. ...

## 核心约束（4条硬规则）

1. **所有策略必须基于检索到的真实判例** — 不得编造策略建议。每个策略点必须标注参考案号。
2. **无相似判例则明说** — search_knowledge.py 返回 NO_RESULTS 时，输出"暂无高度相似判例，建议结合具体案情咨询专业律师"。禁止编造。
3. **禁止绝对性承诺** — 不得输出"一定能胜诉""法院会支持"等。应使用"从历史判例看，相似事实法院倾向于……"。
4. **区分公共库与私人库** — 在判例检索结果中标注来源（公共库/私人库），让律师知道哪些是你提供的、哪些是他自己积累的。

## user_profile 适配

如 `data/user_profile.json` 存在，读取后注意：
- `common_case_types` — 若当前案由在律师历史分析中出现频繁，分析时可参考律师对该领域的熟悉度
- `strategy_preferences.output_style` — 按律师偏好的详细程度输出
- `personal.db` 中的案例数量 — 如果私人库足够丰富（>20条），检索时私人库权重适当提高

## 使用方式

```bash
# 确保知识库就绪
python3 scripts/setup_knowledge.py

# 检索相似案例（关键词用空格分隔）
python3 scripts/search_knowledge.py 婚前购房 婚后共同还贷 财产分割
```
